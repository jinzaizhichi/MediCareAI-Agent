# 🚀 部署指南

---

## 1. 标准部署流程

### 自检仪式

每次执行命令前:
```
- [ ] 没有直接修改生产环境？
- [ ] 本地代码已修改？
- [ ] 修改内容已确认？
```

### 步骤

```bash
# 1. 本地修复
# 在 ~/MediCareAI-Agent/ 修改代码

# 2. 编译检查
python3 -c "compile(open('backend/app/services/orchestrator.py').read(),'o','exec');print('OK')"
cd frontend && npx tsc --noEmit

# 3. Git Commit
git add -A
git commit -m "type(scope): description"

# 4. Git Push
echo '#!/bin/bash\necho "zhanxiaopi"' > /tmp/askpass.sh
chmod +x /tmp/askpass.sh
SSH_ASKPASS=/tmp/askpass.sh DISPLAY=dummy git push origin main

# 5. VPS Pull
ssh root@47.108.235.30 "cd /opt/MediCareAI-Agent && git fetch origin && git reset --hard origin/main"

# 6. Build & Deploy (顺序构建！)
# 全量:
ssh root@47.108.235.30 "cd /opt/MediCareAI-Agent && docker system prune -f && docker compose build --no-cache backend && docker compose build --no-cache frontend && docker compose up -d --force-recreate"

# 仅前端:
ssh root@47.108.235.30 "cd /opt/MediCareAI-Agent && docker compose build --no-cache frontend && docker compose up -d --force-recreate frontend"

# 仅后端:
ssh root@47.108.235.30 "cd /opt/MediCareAI-Agent && docker compose build --no-cache backend && docker compose up -d --force-recreate backend celery-worker celery-beat"

# 7. Verify
ssh root@47.108.235.30 "docker ps --format 'table {{.Names}}\t{{.Status}}'"
curl -s -o /dev/null -w "%{http_code}" https://openmedicareagent.online/
```

```bash
ssh root@47.108.235.30 "docker exec medicareai-postgres psql -U postgres -d medicareai -c \"
SELECT id, session_type, status, jsonb_array_length(context->'lab_reports') as lab_count,
       created_at FROM agent_sessions ORDER BY created_at DESC LIMIT 10;\""
```


## 6. 构建规则

1. **绝不并行构建** — Backend → Frontend 顺序
2. **部署前清理** — `docker system prune -f`
3. **每次部署必须验证** — 健康检查 + HTTP 200
4. **Admin 界面不由 Agent 测试** — 由管理员完成
